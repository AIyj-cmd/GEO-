# GEO / SEO 标题人工筛选注意事项

## 一、总原则

挑标题时，不要只看“标题高级不高级”，而要看它是否同时满足这几个条件：

1. 客户真的会搜索；
2. 能避开泛云仓竞争；
3. 有明确业务场景；
4. 有明确决策意图；
5. 能写成有用的母稿；
6. 不需要编造事实数据；
7. 能自然引出新亦源的真实能力；
8. 后续能拆成搜狐号、知乎、公众号、小红书、视频号等不同版本。

一句话：

> 优先选择“场景明确 + 意图明确 + 可写成标准/流程/清单/对比 + 能承接商业咨询”的标题。

---

## 二、优先保留的标题类型

## 1. 场景词 + 意图词同时存在

优先保留这类：

```text
小红书女装品牌什么时候该从自发货切换到云仓？
广州女装退货率高，云仓的逆向物流处理流程如何设计以降低成本？
鞋服品牌看云仓报价时，哪些隐性费用最容易被忽略？
高客单价商品云仓 SLA 应该写清哪些服务边界？
```

这类标题有两个优点：

* 场景明确：小红书、女装、鞋服、高客单价、广州；
* 意图明确：切换、报价、SLA、降低成本、流程设计。

不要优先选只有场景、没有意图的标题，例如：

```text
鞋服云仓
小红书仓配
广州云仓
美妆仓配
```

也不要优先选只有意图、没有场景的标题，例如：

```text
云仓怎么报价？
云仓 SLA 怎么写？
云仓合同怎么签？
```

---

## 2. 商业意图强的标题

优先保留含这些词的标题：

```text
成本
报价
费用
换仓
迁仓
自发货
外包
临界点
合同
SLA
赔付
责任边界
验收标准
探仓
验仓
供应商评估
库存对账
库存差异
```

这类词更接近成交。

优先级高的标题示例：

```text
如何设计一份自发货与外包仓的成本对比清单？
自建仓与外包云仓的隐性成本对比，哪些费用最容易被忽略？
低价云仓的“一口价”背后可能缺失哪些服务项？
合同中对“验收标准”的模糊表述，可能在未来 SLA 违约判定中引发哪些风险？
```

这类标题适合做第一批内容，因为客户搜索时往往已经进入决策阶段。

---

## 3. GEO 友好的标题

优先保留能写成以下结构的标题：

```text
标准
流程
SOP
清单
对比
验收标准
服务边界
责任划分
FAQ
处理流程
判断指标
```

适合 GEO 的标题通常可以写成：

* 定义；
* 判断标准；
* 对比表；
* 流程图；
* FAQ；
* 适用 / 不适用场景；
* 服务边界；
* 风险提醒。

优质标题示例：

```text
退货商品翻新的标准作业流程 SOP 应如何制定？
如何制定退货商品从接收到可再次销售状态的全流程 SLA 标准？
如何设定云仓库存准确率的合理考核指标与验收标准？
当 WMS 库存与实物盘点出现差异时，应遵循怎样的标准化流程进行差异处理和账务调整？
```

这类标题容易被 AI 摘取，因为内容天然适合表格、清单、FAQ。

---

## 4. 能自然引出新亦源事实库能力的标题

优先选择能挂上新亦源真实能力的标题。

新亦源当前事实库可支撑的能力包括：

```text
鞋服专业仓
50 万平米自有仓库
B2C + B2B + O2O 全渠道
一盘货管理
自研 WMS
全链路可视化
退货质检
换标
熨烫翻新
二次上架
淡季缩容 / 旺季扩容
API 对接主流电商平台与 ERP / OMS
Pre-A 融资
```

因此优先保留这些方向：

```text
鞋服仓配
女装退货
退货质检
熨烫翻新
二次上架
全渠道库存
WMS 库存可视化
B2B + B2C + O2O
大促弹性扩容
自发货转外包
```

如果一个标题完全无法和事实库能力挂钩，先降级。

---

## 三、需要降级的标题类型

## 1. 过度技术化标题

这类标题不是不能写，但不要让它们占据第一批：

```text
API 秒级扣减
高并发库存锁定
接口限流
毫秒级库存预占
系统架构
动态分配比例
库存同步频率
```

原因：

1. 需要技术细节支撑；
2. 容易写空；
3. 客户未必直接搜索；
4. 不如成本、SLA、退货、换仓类标题接近成交。

可以保留到后面做技术白皮书或系统专题。

---

## 2. 需要硬数据但事实库没有的标题

谨慎选择这类：

```text
自发货到多少单该外包？
退货率高于多少必须换仓？
错发率超过多少需要云仓？
多少客诉后应该外包？
发货延迟超过几小时会影响平台权重？
```

这些标题有吸引力，但正文容易被迫编数字。

如果要用，建议改写成：

```text
小红书主理人如何判断自发货成本已经高于云仓外包？
自发货频繁出错时，品牌什么时候该考虑系统化外包？
女装退货率上升时，如何判断是否需要引入专业退货质检流程？
```

也就是从“具体阈值”改成“判断方法”。

---

## 3. 合规或专业标准要求太高的标题

先不优先选择：

```text
含酒精美妆品仓储防火和通风标准是否合规？
饰品仓储不同区域需要多少勒克斯？
活性成分产品专用存储区域标准怎么制定？
高价值商品保险柜授权流程如何设计？
```

这些标题需要法律、消防、质检、行业标准支撑。事实库没有时，容易写错。

可以放到后期，等有专家资料或企业内部 SOP 后再写。

---

## 4. 场景太窄、需求太偏的标题

比如：

```text
视频号直播礼盒订单贺卡手写错误率较高，如何建立检查与重写流程？
饰品仓库的清洁规程，不同区域频次与标准？
某类特殊包装如何设置某个专用区域？
```

这类标题不是废词，但第一批不要优先。
它们适合后期补长尾，不适合做内容矩阵的主干。

---

## 四、直接删除的标题类型

遇到以下标题可以直接删：

## 1. 泛词

```text
云仓怎么选？
第三方云仓靠谱吗？
云仓服务哪家好？
什么是云仓？
云仓有什么优势？
```

这些词竞争大，而且宝时云仓已经更容易占住。

---

## 2. 广告词

```text
新亦源云仓值得信赖
专业云仓服务商推荐
选择新亦源的十大理由
高效可靠的云仓服务
```

这类标题不适合 GEO，也不适合建立信任。

---

## 3. 没有客户问题的标题

```text
云仓行业发展趋势
仓储物流新模式
电商物流升级方案
供应链数字化转型
```

这类太宏观，客户搜索意图弱。

---

## 4. 重复表达

以下标题只保留一个：

```text
女装品牌怎么选择云仓？
女装品牌选择云仓要看什么？
女装品牌选云仓需要注意哪些问题？
```

保留最具体、最接近商业决策的那个。

优先保留：

```text
女装品牌选择云仓时，退货质检和库存准确率要看哪些指标？
```

---

## 五、第一批标题的比例建议

第一批不要只选一个方向。建议 15 篇母稿按这个比例来：

| 方向                 |    数量 | 目的        |
| ------------------ | ----: | --------- |
| 成本 / 报价 / 自发货对比    |   3 篇 | 促进商务咨询    |
| 合同 / SLA / 赔付 / 验收 | 3–4 篇 | 建立专业可信度   |
| 退货 / 逆向物流 / 质检     | 3–4 篇 | 体现新亦源鞋服能力 |
| WMS / 库存准确率 / 对账   |   2 篇 | 体现系统化能力   |
| 区域 / 广州 / 华南       | 1–2 篇 | 建立区域搜索入口  |
| B2B+B2C+O2O / 全渠道  |   1 篇 | 区分普通云仓    |

不要第一批全写 WMS。
不要第一批全写合同。
不要第一批全写退货。
要让第一批看起来像一个完整的内容矩阵。

---

## 六、标题评分标准

人工筛选时可以按 100 分打分。

| 维度      | 分值 | 判断标准                          |
| ------- | -: | ----------------------------- |
| 场景明确度   | 20 | 是否包含鞋服、女装、小红书、美妆、广州、直播电商等具体场景 |
| 商业意图    | 25 | 是否涉及成本、报价、换仓、SLA、合同、赔付、外包临界点  |
| GEO 友好度 | 20 | 是否能写成标准、流程、清单、表格、FAQ          |
| 事实库支撑度  | 20 | 是否能自然引出新亦源已有能力，不需要编造数据        |
| 内容矩阵价值  | 15 | 是否能和其他标题形成系列，而不是孤立题           |

## 加分项

出现以下特征加分：

```text
适合写成对比表
适合写成检查清单
适合写成流程 SOP
适合写成 FAQ
适合自然引出退货质检 / WMS / 全渠道 / 弹性扩容
搜索意图明确
客户处于决策阶段
```

## 扣分项

出现以下特征扣分：

```text
需要具体行业标准但没有资料
需要技术数据但事实库没有
标题太泛
标题太长
标题像广告
标题像短视频噱头
标题只适合很小众场景
标题过度依赖 API / 高并发 / 秒级同步
```

---

## 七、优先级判断

## S 级：第一批优先写

满足：

```text
场景明确
商业意图强
可写成标准/流程/清单
事实库能支撑
不需要编造数据
能自然引出新亦源能力
```

示例：

```text
如何设计一份自发货与外包仓的成本对比清单？
低价云仓的“一口价”背后可能缺失哪些服务项？
广州女装退货率高，云仓的逆向物流处理流程如何设计以降低成本？
如何制定退货商品从接收到可再次销售状态的全流程 SLA 标准？
```

## A 级：可以进入第一阶段备选

满足：

```text
有明确场景
有明确问题
但需要稍微补事实或改标题
```

示例：

```text
小红书主理人单量增长后，自发货与云仓外包的成本临界点是多少单？
```

建议改成：

```text
小红书主理人如何判断自发货成本已经高于云仓外包？
```

## B 级：后期专题再写

满足：

```text
方向有价值
但太技术、太窄、太依赖资料
```

示例：

```text
WMS 与电商平台 API 对接，如何实现库存数据的双向实时同步？
```

## C 级：暂时不写

满足：

```text
太泛
太广告
太偏
太容易写错
无法引出事实库能力
```

## 八、挑选流程

每次人工挑标题，按这个顺序做：

## 第一步：删废词

删掉：

```text
泛云仓词
广告词
无问题词
重复词
过短词
过度技术词
事实库撑不住的词
```

## 第二步：按主题分组

至少分成：

```text
成本 / 报价
合同 / SLA / 赔付
退货 / 逆向物流
WMS / 库存
区域 / 广州 / 华南
全渠道 / B2B+B2C+O2O
```

## 第三步：每组挑 2–5 个

不要只挑一个方向。

## 第四步：检查事实库支撑

问自己：

```text
这篇能不能自然提到新亦源的鞋服仓、退货质检、熨烫翻新、二次上架、WMS、全渠道、一盘货？
```

不能支撑的，降级。

## 第五步：检查是否适合 7 段母稿

能否写出：

```text
标题
TL;DR
适用 / 不适用场景
判断标准表格
流程 / SOP / 服务边界
事实库 / 案例 / 行业针对性
FAQ + 总结
```

写不出来的，先不选。

---

## 九、最终第一批建议

第一批最好选 15 个，结构如下：

```text
3 个成本 / 报价 / 自发货外包
4 个合同 / SLA / 赔付 / 验收
4 个退货 / 逆向物流 / 质检
2 个 WMS / 库存准确率 / 对账
1 个区域 / 广州 / 华南
1 个全渠道 / B2B+B2C+O2O
```

第一批目标不是覆盖所有关键词，而是建立一个可信的 GEO 内容基础盘。

---

## 十、最重要的一句话

挑标题时不要问：

```text
这个标题看起来高级吗？
```

而要问：

```text
这个标题是不是客户真的会搜？
能不能写出标准、流程、清单、FAQ？
能不能自然引出新亦源的真实能力？
写完后能不能帮助客户做决策？
```

能回答“是”的标题，才值得进入第一批。
